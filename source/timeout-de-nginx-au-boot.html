Timeout de NGINX au boot

Avant de passer sous GitHub Pages, ce blog a fait un malencontreux passage sur un VPS SSD 1 d’OVH où se produisait un problème singulier : NGINX ne démarrait pas au boot. 

nginx.service: start operation timed out. Terminating.nginx.service: Failed with result 'timeout'.Failed to start A high performance web server and a reverse proxy server.

Le timeout est donné par la valeur de TimeoutStartSec qui est de 90 secondes par défaut, ce qui est censé être largement suffisant ! De plus, l’augmenter jusqu’à trois minutes ne change rien ; ce n’est probablement pas l’origine du problème.

Un indice se trouve plus loin dans les logs : crng init done.
