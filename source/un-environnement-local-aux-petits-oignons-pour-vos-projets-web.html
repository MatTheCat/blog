Un environnement local aux petits oignons pour vos projets web

Le repository

Si vous êtes seul à travailler sur un projet constitué d’une unique application, vous pouvez vous simplifier la vie et intégrer l’environnement local au dépôt de celle-ci. Sinon, un dépôt dédié (appellons-le local-environment) offrira moult avantages :

  • L’intention est évidente. Dans une liste de dépôts, on saura immédiatement lequel cloner pour commencer à travailler.
  • La gestion de l’environnement est centralisée. Si plusieurs applications composent le projet, on n’a pas à se demander laquelle gère quoi et comment.
  • L’administration est facilitée. Des règles et permissions distinctes peuvent être mises en place pour chaque dépôt.
  • L’environnement et les applications sont séparés. Les sources et documentation de l’un ne se mélangent pas avec celles des autres.

Ceci étant dit, l’environnement local aura bien entendu besoin des sources des applications afin de pouvoir les servir, et il aura également besoin de savoir où les trouver. Si on en a qu’une, on pourra la placer à la racine du dépôt. Sinon, on pourra les regrouper sous un même dossier applications par exemple :

  • local-environment
    • applications
      • foo
      • bar
      • …

Si vous êtes parti sur un dépôt dédié à votre environnement, vous vous demandez peut-être comment y inclure ceux de vos applications. Ceci est rendu possible par Git grâce aux sous-modules :

Les sous-modules vous permettent de gérer un dépôt Git comme un sous-répertoire d’un autre dépôt Git.
